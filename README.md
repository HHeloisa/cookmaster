# Boas vindas ao repositório do projeto Cookmaster!

Este projeto contem o desenvolvimento back-end de um app de gerenciamento de receitas, que utiliza arquitetura MSC (Models, Service e Controllers).
Através dessa aplicação, é possível realizar operações básicas em um determinado banco de dados (MongoDB): Criação, Leitura, Atualização e Exclusão, além da busca por todas as receitas cadastradas, e por ID.
Para realizar qualquer tipo de alteração no banco de dados (como cadastro, edição ou exclusão de receitas) será necessário autenticar-se (via JWT). Além disso, as pessoas usuárias podem ser clientes ou administradores. Pessoas clientes apenas poderão disparar ações nas receitas que ele mesmo criou. Já uma pessoa administradora pode disparar qualquer ação em qualquer receita.
É possível adicionar uma imagem à uma receita já criada, utilizando o upload de arquivos fornecido pelo `multer`.
A aplicação conta com testes de integração, cobrindo mais de 90% das linhas de código.

---

# Habilidades

- Entender o que há por dentro de um token de autenticação;

- Gerar tokens a partir de informações como login e senha;

- Autenticar rotas do Express, usando o token JWT;

- Fazer upload de arquivos em APIs REST;

- Salvar arquivos no servidor através de uma API REST;

- Consultar arquivos do servidor através de uma api REST.

- Realizar testes de integração

# Instalação e uso

- Clone o repositório
- Entre na pasta do repositório que você acabou de clonar
- Instale as dependências
```bash 
  npm install
```
-Inicie a aplicação
```bash 
  npm start
```
- Rode os testes
```bash 
  npm run test
```
- Rode os testes de integração
```bash 
  npm run dev:test
```

### 1 - Crie um endpoint para o cadastro de usuários

- A rota deve ser (`/users`).

- No banco um usuário precisa ter os campos Email, Senha, Nome e Role.

- Para criar um usuário através da API, todos os campos são obrigatórios, com exceção do Role.

- O campo Email deve ser único.

- Usuários criados através desse endpoint devem ter seu campo Role com o atributo _user_, ou seja, devem ser usuários comuns, e não admins.

- O body da requisição deve conter o seguinte formato:

  ```json
  {
    "name": "string",
    "email": "string",
    "password": "string"
  }
  ```
- Não use `bcrypt` ou outra biblioteca para encriptar a senha, para que o avaliador funcione corretamente.

**Além disso, as seguintes verificações serão feitas:**
